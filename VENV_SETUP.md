"""
===================================
虚拟环境配置指南
===================================

功能：
1. 创建 Python 虚拟环境，避免系统 Python 依赖冲突
2. 在虚拟环境中安装项目依赖
3. 激活虚拟环境并运行脚本
"""

# ====================================
# 虚拟环境创建步骤
# ====================================

# 1. 创建虚拟环境
# python -m venv venv

# 2. 激活虚拟环境
# Windows:
# venv\Scripts\activate
# Linux/Mac:
# source venv/bin/activate

# 3. 安装依赖
# pip install -r requirements.txt

# 4. 运行脚本
# python scripts/hs_mainboard_short_strategy.py

# ====================================
# 虚拟环境优势
# ====================================

# 1. 隔离性：每个项目有独立的 Python 环境
# 2. 依赖管理：避免不同项目之间的依赖冲突
# 3. 可移植性：可以轻松复制整个项目到其他机器
# 4. 版本控制：可以精确控制每个包的版本

# ====================================
# 常用命令
# ====================================

# 查看已安装的包
# pip list

# 查看包的详细信息
# pip show <package_name>

# 升级包
# pip install --upgrade <package_name>

# 卸载包
# pip uninstall <package_name>

# 导出依赖列表
# pip freeze > requirements.txt

# ====================================
# 注意事项
# ====================================

# 1. 不要将 venv 目录提交到 Git 仓库
# 2. 在 .gitignore 中添加 venv/
# 3. 定期更新虚拟环境以获取安全补丁
# 4. 使用 requirements.txt 管理项目依赖
