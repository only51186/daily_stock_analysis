# Trae 指令触发式个股分析功能使用说明

## 功能概述

本功能接收以「Trae：」开头的指令，自动解析指令、抓取个股数据、完成超短线分析，并返回结构化回复。

## 支持的指令格式

### 1. 基础格式
```
Trae：分析个股 [股票代码]
```
**示例**：`Trae：分析个股 600000`

### 2. 简化格式
```
Trae：分析个股
```
**说明**：无代码时自动提示「请补充股票代码，示例：Trae：分析个股 600000」

### 3. 多代码格式
```
Trae：分析个股 [股票代码1] [股票代码2] ...
```
**示例**：`Trae：分析个股 600000 000001`

## 分析维度

### 1. 基础行情
- 股票代码/名称/当前股价
- 所属板块/最新涨跌幅
- 换手率/量比/成交额

### 2. 超短线筛选
- 股价是否在5-35元
- 换手率3%-10%
- 量比>1.5
- 5日线支撑

### 3. 情绪分析
- 所属板块热度排名
- 个股近5日涨跌幅趋势
- 涨停基因（近30天涨停次数）

### 4. 预测与建议
- 明日上涨概率（0-100分）
- 买入参考价
- 止损价（下跌3%）
- 持有周期（1-2天）

## 技术实现

### 核心文件
- `scripts/trae_stock_analyze.py` - 主要功能实现
- `test_traecmd.py` - 测试脚本

### 主要类和函数

#### TraeStockAnalyzer 类
- `parse_command()` - 解析指令
- `analyze_stock()` - 分析单个股票
- `format_response()` - 格式化回复
- `process_command()` - 处理指令

#### 便捷函数
- `handle_traecmd(command)` - 处理Trae指令的便捷函数
- `get_analyzer()` - 获取分析器实例

## 配置要求

### 1. 依赖库
- pandas
- numpy
- akshare/efinance（数据源）
- 项目已有的数据提供者模块

### 2. 数据源配置
- 确保 `data_provider` 模块配置正确
- 数据源优先级：akshare → efinance → tushare

### 3. 缓存配置
- 数据缓存有效期：1小时
- 缓存文件存储在 `data_cache` 目录

## 使用方法

### 方法一：直接调用函数

```python
from scripts.trae_stock_analyze import handle_traecmd

# 处理指令
command = "Trae：分析个股 600000"
response = handle_traecmd(command)
print(response)
```

### 方法二：集成到现有系统

1. 在消息处理函数中添加以下代码：

```python
def handle_message(message):
    # 检查是否为Trae指令
    if message.startswith('Trae：'):
        from scripts.trae_stock_analyze import handle_traecmd
        response = handle_traecmd(message)
        if response:
            return response
    # 其他消息处理逻辑
    return ""
```

### 方法三：独立运行测试

```bash
python test_traecmd.py
```

## 性能指标

- **响应时间**：≤3秒
- **Token消耗**：单条分析≤800，批量分析≤2000
- **缓存机制**：本地数据缓存1小时，避免重复请求
- **异常处理**：完善的错误提示和容错机制

## 注意事项

1. 仅分析沪深主板股票（沪市60开头，深市000开头）
2. 仅分析5-35元价格范围的股票
3. 数据来源可能存在延迟，仅供参考
4. 投资有风险，入市需谨慎
