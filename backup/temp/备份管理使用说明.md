# Backup Management User Guide

## Overview

This document describes the backup management system for the Daily Stock Analysis project.

## Backup Naming Convention

```
daily_stock_analysis_backup_YYYYMMDDHHMM.zip
```

- **YYYY**: Year (4 digits)
- **MM**: Month (2 digits)
- **DD**: Day (2 digits)
- **HH**: Hour (24-hour format, 2 digits)
- **MM**: Minute (2 digits)

**Example**: `daily_stock_analysis_backup_202603040056.zip`
- Created on: 2026-03-04 at 00:56

## Backup Location

All backups are stored in:
```
./backup/
```

## Retention Policy

- **Maximum backups retained**: 5
- **Auto-cleanup**: Older backups are automatically deleted when a new backup is created
- **Cleanup trigger**: After each new backup creation

## Excluded Items

The following items are excluded from backups to reduce size:
- `__pycache__/` directories
- `.git/` directory
- `.venv/` and `venv/` virtual environments
- `*.pyc`, `*.pyo` compiled Python files
- `*.log` log files
- `backup/` directory itself
- `openbb_backup/` GitHub backup (2.29 GB)
- `openbb_deps/` dependencies
- `data/` database files
- `logs/` log directory
- `*.db` database files

## Usage Commands

### 1. Create a Manual Backup

```bash
.venv\Scripts\python.exe -c "from utils.backup_manager import get_backup_manager; m = get_backup_manager(); m.create_backup('manual')"
```

### 2. View Backup List

```bash
.venv\Scripts\python.exe -c "from utils.backup_manager import get_backup_manager; m = get_backup_manager(); m.print_backup_list()"
```

**Output Example**:
```
============================================================
Backup List (Latest 5)
============================================================
1. daily_stock_analysis_backup_202603040056.zip
   Time: 2026-03-04 00:56
   Size: 77.19 MB
============================================================
```

### 3. Restore from Backup

**By Timestamp**:
```bash
.venv\Scripts\python.exe -c "from utils.backup_manager import get_backup_manager; m = get_backup_manager(); m.restore_backup('202603040056')"
```

**Important Notes**:
- A pre-restore snapshot is automatically created before restoration
- This ensures the operation can be rolled back if needed

### 4. Get Backup Info by Index

```bash
.venv\Scripts\python.exe -c "from utils.backup_manager import get_backup_manager; m = get_backup_manager(); print(m.get_backup_by_index(1))"
```

## Programmatic Usage

```python
from utils.backup_manager import get_backup_manager

# Initialize manager
manager = get_backup_manager()

# Create backup
result = manager.create_backup(reason="manual")
print(f"Backup created: {result['path']}")

# List backups
backups = manager.list_backups()
for backup in backups:
    print(f"{backup['filename']} - {backup['datetime']} - {backup['size']}")

# Restore from backup
result = manager.restore_backup('202603040056')
print(f"Restore result: {result}")
```

## Backup Workflow

```
┌─────────────────────────────────────────────────────────────┐
│                    Backup Creation Flow                      │
├─────────────────────────────────────────────────────────────┤
│  1. Generate timestamp (YYYYMMDDHHMM)                       │
│  2. Create zip archive (excluding large directories)        │
│  3. Save to ./backup/ directory                             │
│  4. Check backup count                                      │
│  5. If > 5 backups, delete oldest ones                      │
│  6. Return backup info                                      │
└─────────────────────────────────────────────────────────────┘
```

## Restore Workflow

```
┌─────────────────────────────────────────────────────────────┐
│                    Restore Operation Flow                    │
├─────────────────────────────────────────────────────────────┤
│  1. Create pre-restore snapshot (safety backup)             │
│  2. Locate target backup by timestamp                       │
│  3. Extract zip archive to project root                     │
│  4. Overwrite existing files                                │
│  5. Return restore result with pre-restore backup path      │
└─────────────────────────────────────────────────────────────┘
```

## Safety Features

1. **Pre-restore Backup**: Before any restore operation, a snapshot is automatically created
2. **Rollback Support**: If restore fails or produces unexpected results, use the pre-restore backup
3. **Exclusion Rules**: Large files and generated content are excluded to keep backups manageable
4. **Auto-cleanup**: Prevents disk space overflow by limiting backup count

## Troubleshooting

### Issue: Backup creation fails

**Possible causes**:
- Insufficient disk space
- File permission issues
- Files locked by other processes

**Solution**:
- Check disk space
- Run as administrator if needed
- Close other applications using the files

### Issue: Restore fails

**Possible causes**:
- Backup file corrupted
- Target backup not found
- Insufficient permissions

**Solution**:
- Verify backup file integrity
- Check backup timestamp is correct
- Run as administrator

## Integration with Development Workflow

The backup manager is designed to integrate with the development workflow:

1. **Before major changes**: Create a backup
2. **After successful changes**: Create a backup
3. **Before restoration**: Pre-restore snapshot is automatic
4. **After testing**: Create a backup of working state
